Padrões Inegociáveis de Código

TypeScript strict: true em tudoSem any implícito, sem erros ignorados
Zod em todas as entradas externasWhatsApp webhooks, ISP APIs, uploads
RLS activo em TODAS as tabelasMulti-tenant validado no DB, não só no código
Idempotency keys em pagamentosUUID gerado no front, validado no back
HMAC em todos os webhooksStripe, WhatsApp, ISP callbacks
Circuit Breaker em todas as APIs externasOpenAI, WhatsApp, gateways de pagamento
Outbox Pattern em toda escrita + eventoNunca disparar fila sem gravar no DB antes
PII anonimizado antes de enviar à OpenAIMicrosoft Presidio ou equivalente Node.js

Regras de Escalada (Quando migrar para Bare-Metal)

📊
Gatilho 1 — Custo OpenAI: quando a fatura mensal da API ultrapassar 40% da receita MRR → iniciar migração para vLLM + Llama no Mac Studio / Hetzner.
⚡
Gatilho 2 — Volume de filas: quando BullMQ processar >500k jobs/dia consistentemente → avaliar migração para Redpanda ou NATS JetStream.
🗄️
Gatilho 3 — Analytics: quando DuckDB in-process demorar >3s em queries → migrar para ClickHouse dedicado + TimescaleDB para telemetria de rede.
🔐
Gatilho 4 — Contratos Enterprise: quando ISP exigir SSO / SAML / LDAP → migrar Supabase Auth para Keycloak + Opaque Tokens.
☁️
Gatilho 5 — Cloud bill: quando custo de PaaS ultrapassar custo equivalente de Hetzner × 2 → migrar para K3s em Bare-Metal com ArgoCD GitOps.
